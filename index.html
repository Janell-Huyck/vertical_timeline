Connect Four

Sep 2019

This four-person group project involved recreating the classic board game Connect Four. When a column is clicked, a red or black piece is added to the top of the column, depending on whose turn it is. Winning scenarios are checked after every move for horizontal, vertical, and diagonal.

This was our first attempt at using Agile and coordinating a git repository among multiple developers. Chief among the lessons learned would be how important it is to have everyone with the same understanding of the underlying structure of the project, and how important it is to NOT push a non-functioning branch to the master branch so that all your teammates clone it. LOL!

My role included the overall project design, creating the table, the logic and implementation for placing pieces, the logic and implementation for the win conditions, and quality control.

Kwitter

Dec 2019

This is a micro-blogging platform. This front-end group project built on a pre-made API. We were introduced to Higher Order Components, and used React-Redux to build this project. We used an Agile process and scrum board to coordinate our work.

My role included making the message feed and individual messages, adding a like/unlike function to individual messages, adding the delete function for individual messages, ensuring the function of the edit-profile page, and general quality control.

React | React-Redux | JSX | CSS

Repository

Battleship

Jan 2019

This front-end capstone 7-person group project recreated the classic board game Battleship. We were given three weeks and freedom over the nature and design of the project. We chose to build an API-interacting game that worked off the same database framework we had used for Kwitter. Two players on different computers play against eachother, taking turns firing torpedos against their opponent's locations until one person has sunk all the opponents' ships. Our group included a UX/UI design student, two people who focused almost exclusively on the css, and the remainder worked on the JavaScript.

My roles in this project were the product owner and team leader. Because of the tight time-table, our group allowed me to come up with the overall design and strategy of the project and lay out the react-redux framework before the project start-date. I facilitated daily standup meetings, encouraged all members' participation, answered technical questions and assisted with every step of the project. No one component was exclusively mine, but I participated to some degree in almost everything.
